<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (query string) ‚Üí route handler | `?ccn=` is untrusted user input; gated + normalized before any use |
| route ‚Üí CMS upstream fetch | CCN crosses into an outbound HTTP request as a condition value only |
| CMS response ‚Üí Zod validation | Untrusted upstream JSON; never reaches the consumer unvalidated (rule #4) |
| route ‚Üí client (error body) | Server-authored error payload; must leak no internals (D-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-CCN | Tampering | route CCN param | mitigate | Trim + uppercase-normalize + length-cap, gate on `/^[A-Za-z0-9]{6}$/` before any fetch; reject (400) malformed input so no crafted value reaches the URL (D-22/D-07). |
| T-02-SSRF | Tampering | client.ts CMS URL | mitigate | Host/path come from fixed constants (Plan 02-01); CCN is only a `conditions[0][value]`, never concatenated into host/path; the 6-alphanumeric gate further constrains it (D-24). |
| T-02-LEAK | Information Disclosure | validation_error response body | mitigate | Body is exactly `{ error: { kind: 'validation_error', message } }` ‚Äî no ccn, no Zod paths; `z.prettifyError` goes only to `console.error` (D-05/D-06). Enforced by the leak-invariant test (`/issues\|expected\|received\|path\|code/` must NOT match). |
| T-02-DOS | Denial of Service | upstream CMS latency | mitigate | `AbortSignal.timeout(8000)` caps the wait under Vercel's ~10s function wall ‚Üí clean `network_error` (502) instead of an opaque platform 504 (D-19). |
| T-02-VAL | Tampering | CMS response shape | mitigate | `safeParseCMSRow` (Zod) validates before mapping; a renamed/removed key ‚Üí `validation_error`, never fabricated data (rule #4). |
| T-02-SC | Tampering | npm installs | accept | No new packages installed; all deps legitimacy-gated in Phase 1 (01-01). No package-manager install tasks here. |
</threat_model>
