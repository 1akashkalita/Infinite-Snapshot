<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (POST body) → export route | The client POSTs a full `ReportViewModel`; untrusted, Zod-validated before any use (D-20) |
| view-model → all render targets | The single shared model is the cross-target consistency seam (RPT-02) |
| facility name (manual override) → report | The override is body-only; it must never reach the static header (NAME-02/RPT-01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-POST | Tampering | POST /api/export/pdf body | mitigate | `ReportViewModelSchema.safeParse(body)` rejects any malformed shape with a clean 400 before doing anything (D-20/D-21); the client view model is never trusted. |
| T-02-LEAK2 | Information Disclosure | export-route 400 body | mitigate | The 400 body is exactly `{ error: { kind: 'invalid_request', message } }` — no Zod paths/internals (same discipline as the facility route's D-05). |
| T-02-BRAND | Tampering | static header vs facility name | mitigate | `assembleHeader(state)` takes ONLY a state code (rule #2/RPT-01); the manual `nameOverride` flows to `displayName` (body) only, proven by the NAME-02 isolation test. |
| T-02-BUILD | Tampering | next.config / standalone build | mitigate | `serverExternalPackages: ['@react-pdf/renderer']` (D-25) defends against Turbopack bug #88844 omitting it; verified by a real `next build` in `verify:full`. |
| T-02-SC | Tampering | npm installs | accept | No new packages installed; all five prod libs were legitimacy-gated in Phase 1 (01-01). No package-manager install tasks here. |
</threat_model>
