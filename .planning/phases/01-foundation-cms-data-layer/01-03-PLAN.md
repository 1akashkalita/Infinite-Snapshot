<objective>
Build the provider-only Zod validation layer: `CMSRowSchema` (with the verified empty-string→null→coerce pipeline) plus a typed parse module, and a test suite that proves every required DATA-02 behavior and the DATA-06 traceability invariant against the captured fixture.

Purpose: This is the runtime enforcement of CLAUDE.md rule #4 (every CMS response validated by Zod) and rule #3 / DATA-06 (a renamed/removed CMS key fails loudly instead of silently passing). It is also the single source of clean typed values for Phase 2's `/api/facility` route and `ReportViewModel`.
Output: `src/lib/cms/schema.ts`, `src/lib/cms/parse.ts`, `tests/lib/cms/schema.test.ts`; `npm run verify` green.

IMPORTANT — intentional spec deviation to surface to the verifier (D-06): ROADMAP Phase 1 SC#4 literally says star-rating fields are `.nullable().optional()`. CONTEXT.md D-05/D-06 deliberately REFINE this to required-key + `.nullable()`-value (the `.optional()` is intentionally dropped). This is correct, not a defect: CMS suppression is always an empty STRING with the key present, never a missing key, so dropping `.optional()` is what makes a renamed/removed key fail loudly (the DATA-06 invariant). The verifier MUST NOT flag the absent `.optional()` as a failure of SC#4 — same intent ("a suppressed-data facility does not throw"), enforced more strictly.
</objective>

Empty-string -> null -> coerce helper (verified):
  z.preprocess((v) => (typeof v === "string" && v.trim() === "" ? null : v), z.coerce.number().nullable())
  Behaviors:  ""/"   " -> null   |   "0" -> 0   |   "5" -> 5   |   null -> null

Zod v4 breaking change: on failure use result.error.issues (result.error.errors is UNDEFINED in v4).
z.prettifyError(result.error) returns a human-readable string.

Fields to model in CMSRowSchema (all required keys; names verified live in provider-686123.json):
  Strings (NO coercion — D-10):  cms_certification_number_ccn, zip_code, provider_name,
                                  legal_business_name, provider_address, citytown, state, processing_date
  Numeric (nullableNum helper):  number_of_certified_beds
  Star ratings (nullableNum, D-05/D-06):  overall_rating, health_inspection_rating,

  qm_rating is the Quality of Resident Care rating — NOT longstay_qm_rating / shortstay_qm_rating.
  Terminate the object with .passthrough() (D-04 — ~90 unmodeled CMS columns pass through).
