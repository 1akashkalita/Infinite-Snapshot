## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-PDF | Tampering | hospMetrics in the PDF POST body | mitigate | POST /api/export/pdf re-validates the full posted model against ReportViewModelSchema (Plan 03 Task 1): `unit` is z.enum(["percent","rate"]), `value` is z.number().nullable(), `label`/`footnoteCode` are strings — a crafted body cannot inject executable/arbitrary metric content (D-13). |
| T-05-XSS | Tampering | metric label/value rendered in preview | accept | Labels come from the hardcoded METRIC_DEFINITIONS (not user input) and values are formatted numbers/known footnote strings; React escapes text nodes by default. No raw HTML injection surface. |
| T-05-BUNDLE | Information Disclosure | react-pdf leaking to client bundle | mitigate | ReportPDF stays server-only (no "use client"); SnapshotApp imports HospMetric as a type only and never imports react-pdf/cms-client (T-03-09). `next build` in verify:full fails if react-pdf reaches the client bundle. |
| T-05-SC | Tampering | npm installs | mitigate | No new packages this phase (RESEARCH audit empty). |
</threat_model>
