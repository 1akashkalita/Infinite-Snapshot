<objective>

preview populate from CMS — plus every distinct error state.

populated snapshot (manual inputs come in Wave 4).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CCN input → /api/facility | Untrusted CCN crosses to the server route (the source-of-truth gate) |
| /api/facility → preview render | Typed FacilityData (Zod-validated server-side) crosses; rendered via React JSX |
| facility-name override → body | User string rendered into the body displayName (Wave 4 binds it; rendered here) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | CCN → fetch URL (SnapshotApp.handleSearch) | mitigate | CCN passed only as a query-param value via `encodeURIComponent`, never a path segment; client pre-check + server gate both apply `/^[A-Za-z0-9]{6}$/` |
| T-03-07 | Tampering (XSS) | ReportPreview rendering CMS values + (Wave 4) name override | mitigate | All values rendered via React JSX interpolation (auto-escaped); NO `dangerouslySetInnerHTML` anywhere |
| T-03-08 | Information Disclosure | error rendering in ErrorBanner/CCNSearchBar | mitigate | UI renders only `getErrorPresentation(error).message` (UI-authored copy); the server already strips Zod internals from `validation_error` (Phase-2 D-05) |
| T-03-09 | Tampering | client import tree | mitigate | Components import only `@/lib/report/*`, `@/lib/cms/{errors,types}`, `@/lib/ui/*`; never `@/lib/cms/client` or `@react-pdf/renderer` (RESEARCH Pitfall 1) — `next build` in Task 3 catches a bad import |
| T-03-SC | Tampering | npm installs | accept | No package installs in this plan (Phase 3 adds zero new packages) |
</threat_model>
