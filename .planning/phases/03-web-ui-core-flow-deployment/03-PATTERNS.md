### `src/lib/ui/error-presentation.ts` — error-kind → UI copy (utility, transform)

**Core exhaustive switch pattern** (mirrors route.ts lines 80–121 exhaustive handling; `assertNever` receives the discriminant — `error.kind` not `error`; see RESEARCH.md Pitfall 4):
```typescript
export type ErrorPlacement = 'inline' | 'banner'

export function getErrorPresentation(error: CmsApiError): {
  message: string
  placement: ErrorPlacement
} {
  switch (error.kind) {
    case 'invalid_ccn':
      return { message: 'Please enter a valid 6-character CCN (letters and numbers only).', placement: 'inline' }
    case 'not_found':
      return { message: `No facility found for CCN "${error.ccn}". Check the number and try again.`, placement: 'inline' }
    case 'network_error':
      return { message: 'Could not reach CMS. Check your connection and try again.', placement: 'banner' }
    case 'cms_api_error':
      return { message: 'CMS is temporarily unavailable. Please try again in a moment.', placement: 'banner' }
    case 'validation_error':
      // D-08: non-retry copy — this won't heal on retry
      return { message: 'Received an unexpected response from CMS. This issue has been noted.', placement: 'banner' }
    default:
      return assertNever(error.kind)  // compile error if 6th kind added without case
  }
}
```

## Shared Patterns

### `"use client"` boundary

**Source:** All `src/components/*.tsx` files.
**Apply to:** `SnapshotApp.tsx`, `CCNSearchBar.tsx`, `ManualInputsForm.tsx`, `ReportPreview.tsx`, `ErrorBanner.tsx`.

Rule: The directive `'use client'` must be the **first non-comment line** of the file. Do not add it to `page.tsx` (server component), `layout.tsx`, or any `src/lib/**` module.

```tsx
'use client'
// imports follow
```

### `@/*` path alias

**Source:** `vitest.config.ts` lines 5–7; all existing lib and test imports.
**Apply to:** Every import in `src/components/`, `src/lib/ui/`, and all new test files.

```typescript
// All internal imports use @/* — never relative paths across directory boundaries:
import { assembleViewModel } from '@/lib/report/view-model'
import type { CmsApiError } from '@/lib/cms/errors'
import { assertNever } from '@/lib/cms/errors'
```

### `isolatedModules` export requirement

**Source:** All existing `src/lib/**` files (each has at least one `export`).
**Apply to:** `src/lib/ui/ccn.ts`, `src/lib/ui/error-presentation.ts`, all new component files.

Rule: Every `.ts` / `.tsx` file must have at least one `import` or `export` statement. Pure-logic utility files must `export` their functions (not use default export for tree-shaking friendliness).

### Vitest node-env test file structure

**Source:** `tests/lib/report/format.test.ts` lines 1–16; `tests/lib/cms/errors.test.ts` lines 1–4.
**Apply to:** `tests/lib/ccn-precheck.test.ts`, `tests/lib/error-kind-mapping.test.ts`.

```typescript
import { describe, expect, it } from "vitest";
// Named imports from @/* — never from relative paths out of tests/ root
```

No DOM, no React rendering, no `@testing-library/react`. Only pure-function imports are testable in the `node` environment.
